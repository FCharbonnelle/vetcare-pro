## 2026-03-26 - [Performance Optimization: Nested Component Extraction]
**Learning:** Defining functional components inside the render body of another component is a significant performance anti-pattern. It causes the inner component to be re-created on every render of the parent, leading to full unmount/remount cycles of the entire subtree. This is especially problematic during animations where renders occur at ~60fps. Additionally, `React.memo` is only effective if props (including event handlers) are stable, necessitating the use of `useCallback` for functions passed as props.
**Action:** Always extract sub-components to the top-level of the file. Use `React.memo` for leaf components that render frequently, and ensure props are stable by using `useCallback` for function props and `useMemo` for objects/arrays.
